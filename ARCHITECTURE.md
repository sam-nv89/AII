# Архитектура Платформы AI-Интегратор 2026

Документ описывает техническое устройство и логику работы SPA-приложения.

## 🏗 Общая структура (SPA)

Приложение построено на базе **Single Page Application** модели без использования тяжелых фреймворков. Это обеспечивает мгновенную загрузку и легкость поддержки.

### Основные компоненты системы:
1.  **Router (`js/app.js`)**: Обрабатывает изменения хэша в URL (`#home`, `#module-N`, `#quiz-N`) и динамически перерисовывает содержимое основного блока `<main id="page-content">`.
2.  **State Manager (`js/store.js`)**: Единственный источник истины. Управляет данными пользователя (XP, прогресс тестов, текущий шаг) и синхронизирует их с `localStorage`.
3.  **Modules Data (`js/modules/`)**: Статические данные курса. Каждый файл содержит теорию, массив шагов воркшопа и 10 вопросов теста.
4.  **View Renderers (`js/views/`)**: Функции, которые принимают данные и возвращают HTML-строки для отображения (Home, Модули, Тесты).

## 💾 Схема данных (Store)

Данные хранятся в объекте со следующей структурой:
```javascript
{
  xp: 1250,                 // Общий опыт
  completedModules: [1, 2], // Массив ID пройденных модулей
  quizScores: {             // Проценты верных ответов
    "1": 90,
    "2": 100
  },
  currentModule: 2,         // Модуль, на котором остановился юзер
  startDate: 1713180000000  // Timestamp начала обучения
}
```

## 🎨 Дизайн-система (CSS)

Стилизация разделена на 4 уровня для чистоты кода:
-   `css/reset.css`: Сброс браузерных стилей.
-   `css/tokens.css`: Дизайн-токены (цвета, шрифты, тени, анимационные тайминги).
-   `css/layout.css`: Сетка приложения (сайдбар + основная область).
-   `css/components.css`: Атомарные компоненты (карточки, кнопки, прогресс-бары, инпуты тестов).

## 🧩 Система квизов (Quiz Engine)

Движок тестов в `js/views/quiz.js` работает по циклу:
1.  Инициализация (сброс ответов, запуск таймера).
2.  Отрисовка вопроса (динамическая генерация опций).
3.  Сбор ответов в массив.
4.  Финализация: расчет результата → проверка порога (70%) → начисление XP (через Store) → запуск конфетти.

## 📈 XP и Уровни

Уровни рассчитываются динамически на основе XP:
-   **Уровень 1 (Новичок)**: 0 - 499 XP
-   **Уровень 2 (Младший Интегратор)**: 500 - 999 XP
-   **Уровень 3 (Специалист)**: 1000 - 1999 XP
-   и т.д.

Формула расчета прогресса до следующего уровня: `(XP - текущий_порог) / (следующий_порог - текущий_порог) * 100`.
